# 图书馆新生AI导览助手"吉小图" — 项目说明文档

## 一、项目概述

"吉小图"是一个基于 NVIDIA DGX Spark GB10 和 Stepfun 阶跃星辰 Step 3.7 Flash 构建的**多智能体图书馆AI导览系统**。项目面向吉利学院图书馆真实场景，解决新生入学后"找书难、找路难、规则多记不住"的核心痛点。

**核心功能**：用户通过自然语言提问或上传照片，即可查询图书馆规则、定位图书/区域位置、获取楼层导航路径。

**在线体验**：手机浏览器访问即可，无需安装 APP。

---

## 二、核心亮点

### 1. 多智能体组织化协作架构

区别于传统的"单模型+API调用"模式，本项目设计了**四个独立Agent**，各司其职、边界清晰：

- **Guide Agent（导览Agent）**：负责对话交互和意图分类，是用户唯一的对话入口。不直接查库，只做路由和回复合成。
- **Search Agent（检索Agent）**：负责RAG知识库查询，是知识的唯一入口。不直接对用户说话，只返回检索结果。
- **Planning Agent（规划Agent）**：负责A*路径规划和导航文本生成，不参与对话也不查库，只做空间计算。
- **Verify Agent（校验Agent）**：负责事实核查和幻觉检测。不生成内容，只做事实校验。

四个Agent通过 `AgentContext` 共享会话状态（记忆），通过 `allowed_actions`/`forbidden_actions` 强制执行边界约束，实现了**"有角色、有边界、有记忆"**的组织化协作。借鉴上一届获奖作品"Starfire-AgentTeam"的核心理念。

### 2. 混合推理架构

- **意图分类/校验**：Stepfun Step 3.7 Flash API — 快速轻量调用（1-2秒）
- **文本回复生成**：DGX Spark 本地 Qwen3.6-35B-A3B MoE — GPU推理，35B总参/3B激活参数
- **多模态识别**：Stepfun Step 3.7 Flash — 原生视觉理解，拍照识别书架位置
- **RAG 检索**：BM25 关键词匹配 — 104篇知识库文档，即时响应无网络依赖

### 3. 真实数据驱动的 RAG 知识库

知识库基于**吉利学院图书馆真实数据**构建，包含：
- 53条FAQ（借阅规则、开馆时间、座位预约、馆藏分布、设施使用、新生指南等）
- 25条规章制度（入馆、借阅、阅览、数字资源、违规处理、开馆时间）
- 4层楼的完整空间数据（30+区域节点、80+导航边）
- 支持 MySQL 真实馆藏数据库同步

RAG 管道采用 BM25 关键词检索，每次搜索从磁盘重新加载最新数据，确保知识库实时更新。

### 4. 端到端完整闭环

从用户输入（文本/图片）→ 意图理解 → 知识检索 → 路径规划 → 回复生成 → 事实校验 → 回复展示，形成完整的处理闭环。

系统设计了多层降级策略：
- 本地 LLM 不可用 → Stepfun API 降级
- 图片识别失败 → 预设位置 + 关键词查询降级
- 向量检索不可用 → BM25 关键词降级
- API 超时 → 自动重试 3 次（2s/4s 间隔）

## 三、技术架构

```
手机浏览器 (Mobile-First Web)
       │ HTTP/SSE
       ▼
FastAPI 后端 (DGX Spark GB10, 128GB统一内存)
  ├── Multi-Agent 编排层 (Orchestrator 状态机)
  │   ├── Guide Agent (对话+意图路由)
  │   ├── Search Agent (RAG 知识检索)
  │   ├── Planning Agent (A* 路径规划)
  │   └── Verify Agent (幻觉检测)
  ├── RAG 检索引擎
  │   └── BM25 关键词索引 (104篇文档, 即时加载)
  ├── 模型层
  │   ├── Qwen3.6-35B-A3B MoE (本地 GPU, llama.cpp)
  │   └── Step 3.7 Flash API (多模态+意图+校验)
  └── 数据层
      ├── JSON 知识库 (FAQ + 规则 + 楼层)
      └── MySQL 馆藏数据库 (可选同步)
```

## 五、创新点总结

1. **四Agent协作架构**：每个Agent角色明确、边界清晰，实现了真正的"组织化协作"而非简单的"多模型堆叠"
2. **混合推理**：DGX Spark 本地GPU推理主力文本生成 + Stepfun API 处理多模态和轻量任务，充分利用各自优势
3. **多模态+导航一体化**：拍照→识别→定位→回答，一条链路打通
4. **真实数据落地**：基于学校实际借阅规则和馆藏数据，可直接部署使用
5. **多层降级策略**：从本地LLM到API到关键词搜索，确保系统在任何条件下都能提供基本服务
