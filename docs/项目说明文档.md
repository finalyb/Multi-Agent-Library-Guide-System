# 图书馆新生AI导览助手"吉小图" — 项目说明文档

## 一、项目概述

"吉小图"是一个基于 NVIDIA DGX Spark 和 Stepfun 阶跃星辰 Step 3.7 Flash 模型构建的**多智能体图书馆AI导览系统**。项目面向吉利学院图书馆真实场景，解决新生入学后"找书难、找路难、规则多记不住"的核心痛点。

**核心功能**：用户通过自然语言或拍照即可查询图书馆规则、定位图书位置、获取楼层导航路径。

## 二、核心亮点

### 1. 多智能体组织化协作架构（评审关键）

区别于传统的"单模型+API调用"模式，本项目设计了**四个独立Agent**，各司其职、边界清晰：

- **Guide Agent（导览Agent）**：负责对话交互和意图分类，是用户唯一的对话入口。它不直接查库，只做路由和回复合成。
- **Search Agent（检索Agent）**：负责RAG知识库查询，是知识的唯一入口。它不直接对用户说话，只返回检索结果。
- **Planning Agent（规划Agent）**：负责A*路径规划和导航文本生成，它不参与对话也不查库，只做空间计算。
- **Verify Agent（校验Agent）**：负责事实核查和幻觉检测。它不生成内容，只做"事实警察"。

四个Agent通过 `AgentContext` 共享会话状态（记忆），通过 `allowed_actions`/`forbidden_actions` 强制执行边界约束，实现了**"有角色、有边界、有记忆"**的组织化协作。

### 2. Step 3.7 Flash 的深度应用

本项目不是简单的"调API"，而是在多个维度上深度使用 Step 3.7 Flash 的能力：

- **原生多模态理解**：用户拍照书架/图书封面 → Flash 直接识别索书号标签和区域特征，无需单独的OCR+分类器管线
- **高可靠工具调用**：Agent 通过 function calling 机制调度 RAG 检索和路径规划
- **意图分类**：使用 Flash 的语义理解能力精确判断用户意图（7种意图类型）
- **查询重写**：自动将口语化查询重写为检索友好的形式，提升RAG召回率
- **重排序**：对向量检索结果进行二次LLM打分，确保返回最相关的信息

### 3. 真实数据驱动的RAG

知识库基于**吉利学院图书馆真实MySQL馆藏数据库**构建，包含：
- 50条FAQ（覆盖借阅、开馆、座位、馆藏、设施等）
- 25条规章制度原文
- 4层楼的完整空间数据（30+区域节点，80+导航边）
- 真实馆藏图书数据（通过只读MySQL账号同步）

RAG管道采用**混合检索策略**：ChromaDB向量检索 + BM25关键词检索 → RRF融合排序 → Step 3.7 Flash重排序，确保高召回率和高准确度。

### 4. 端到端完整闭环

从用户输入（文本/图片）→ 意图理解 → 知识检索 → 路径规划 → 回复生成 → 事实校验 → 回复展示，形成完整的处理闭环。系统还设计了**降级策略**：Step 3.7 Flash 不可用时自动回退到关键词匹配，确保基本服务可用。

## 三、技术架构

```
前端 (Mobile-First HTML/CSS/JS)
  ↓ HTTP/SSE
FastAPI 后端
  ├── Multi-Agent 编排层 (Orchestrator)
  │   ├── Guide Agent (对话+意图)
  │   ├── Search Agent (RAG检索)
  │   ├── Planning Agent (A*路径)
  │   └── Verify Agent (幻觉检测)
  ├── RAG 混合检索引擎
  │   ├── ChromaDB 向量库
  │   ├── BM25 关键词索引
  │   └── RRF 融合排序
  ├── 模型层
  │   ├── Step 3.7 Flash (多模态+对话+嵌入)
  │   └── NVIDIA Nemotron (本地后备)
  └── 数据层
      ├── JSON 知识库 (FAQ + 规则 + 楼层)
      └── MySQL 馆藏数据库 (只读)
```

## 四、应用场景

1. **新生入学**：通过拍照导航快速熟悉图书馆各楼层布局
2. **日常咨询**：解答借阅规则、开放时间、座位预约等高频问题
3. **图书查找**：输入书名获取精确的楼层-区域-书架位置
4. **路径导航**：从当前位置到目标位置的分步文字指引

## 五、创新点总结

1. **四Agent协作架构**：每个Agent角色明确、边界清晰，实现了真正的"组织化协作"而非简单的"多模型堆叠"
2. **多模态+导航一体化**：拍照→识别→定位→导航，一条链路打通
3. **真实数据落地**：对接学校馆藏MySQL数据库，不是"玩具Demo"而是可实际部署的系统
4. **完整的工程实践**：包含降级策略、流式响应、会话管理、幻觉检测等生产级特性
5. **Stepfun + NVIDIA 双模型协同**：Step 3.7 Flash 负责核心推理，Nemotron 负责校验和后备，充分发挥两者优势
