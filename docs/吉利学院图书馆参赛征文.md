## 七日交卷

赛程给了十天（7月12日—7月22日）。写下这篇征文的今天是 7月18日——第七天，"吉小图"已经在 DGX Spark（spark-78）上完整跑起来：四个 Agent 各司其职，本地 GPU 和云端 API 协同工作，拍照识别和知识检索无缝衔接。

对照完赛标准后，我们决定今天交卷。

## Day 0：方向选择 — "图书馆+AI"的天然契合

**7月12日 · 训练营启动日**

接到 Hackathon 题目时，团队第一反应是"做什么能既体现技术深度，又有真实落地价值？"

我们来自吉利学院图书馆，每天都在面对新生问路、找书、问规则。没有比这更真实的场景了——**图书馆本身就是一个天然的知识库+服务场景**。

"图书馆新生AI导览助手"几乎没有争议就通过了。三个理由：
1. 我们懂这个场景（团队每天看着馆员重复回答同样的问题）
2. 我们有真实数据基础（馆藏数据库、规章制度、楼层平面图）
3. Demo极度直观（拍照→识别→导航，评委3秒看懂）

方向定了就不摇摆。当天下午云节点开放，我们拿到了 spark-78；当晚，38 个文件的项目骨架立了起来——四 Agent 框架、RAG 管道、前端界面、知识库数据一次成型，Step 3.7 Flash 的 API 也在睡前验证通过（还顺手用上了 Stepfun 独有的 `reasoning_effort` 参数）。

Day 0 结束时，我们已经不是从零开始了。

## Day 1-2：核心跑通 — 从零到第一个回答

**7月13-14日**

骨架好搭，血肉难长。这两天几乎全耗在 `stepfun_client.py` 的调试上——Step 3.7 Flash 是整个系统的发动机，OpenAI-compatible 协议不算难，但多模态输入格式踩了不少坑：图片 base64 编码、content 数组结构、temperature 参数选择……每一步都试错。

Day 2 晚上，第一个"你是谁"成功返回了正确回答。系统随之部署上云主机，公网可以访问了。

> ⚠️ **关于数据**：正值暑期放假，图书馆处于闭馆状态，因此本项目数据目前为**测试数据集**（53 条 FAQ + 25 条规则 + 4 层楼面，共 104 篇文档），真实部署时可通过 MySQL 同步接口一键切换为生产数据。

## Day 3-4：双线作战 — Agent 边界与本地算力

**7月15-16日**

这两天是整个开发周期强度最高的，两条线并行。

**第一条线：Agent 架构。**看了往届获奖作品（尤其是"Starfire-AgentTeam"）后，我们意识到评审想看的是**"组织化协作"**，不是"单兵作战"。于是有了四个Agent的设计：

- Guide Agent → 只管对话和意图路由
- Search Agent → 只管RAG检索
- Planning Agent → 只做A*路径计算
- Verify Agent → 只做事实核查

每个Agent有明确的 `allowed_actions` 和 `forbidden_actions` 边界约束。**边界比能力更重要**——这是我们对多智能体系统最深的体会。

**第二条线：把 DGX Spark 的算力真正用起来。**GB10 拥有 128GB 统一内存，光用来调 API 太浪费了。我们在云主机上编译了 llama.cpp（CUDA 13.0，sm_120 架构），从 ModelScope 国内源下载了 Qwen3.6-35B-A3B MoE 的 MXFP4 量化模型（~21GB）。模型加载 9 秒，推理速度约 43 tok/s。

最终采用**混合路由架构**：轻量任务（意图分类、校验）和多模态走 Stepfun API，知识问答的回复生成走本地 GPU——充分利用平台算力，同时保证响应速度。

## Day 5：多模态高光与细节打磨

**7月17日**

Step 3.7 Flash 的原生多模态能力远超预期。给出期刊阅览区照片，模型直接返回位置描述——不需要单独的 OCR、不需要 YOLO 检测，Flash 直接"看懂"了。**拍照→识别→定位→回答**，一条链路打通，这是 Demo 的高光时刻。

高光背后是一整天的细节修补，每一条都是真实的坑：

- **身份问题**：本地 Qwen 模型默认回答"我是通义千问"——闹了个笑话。通过路由修复强制闲聊走 Stepfun，确保始终回应"我是吉小图"
- **RAG 检索**：嵌入模型因云主机网络限制下载不了，果断切换为 BM25 关键词匹配，每次从磁盘重载数据确保实时性——工程上"能稳定跑通的方案"永远优于"理论上更优的方案"
- **拍照上传**：浏览器相机权限在部分设备上不稳定，改为文件上传，更稳更通用
- **IP 形象**：设计了吉小图专属头像替换默认机器人图标
- **借阅规则**：按学校实际政策更新（学生教师各20册、借期30天、可续借1次，开馆 7:30-22:00）

## Day 6：完赛日 — 对照基线，交卷

**7月18日**

微信群明确了基础完赛标准——以训练营 Workshop 讲义（本地大模型 + Agent 平台 + 自定义技能 + 端到端多模态闭环）为基线。我们把"吉小图"逐项对照自检：

| 基线能力 | 吉小图的实现 |
|---|---|
| 本地大模型部署 | llama.cpp + Qwen3.6-35B MXFP4，本地 GPU 推理 |
| Agent 化与能力扩展 | 自研四 Agent 协作体系（角色、边界、记忆） |
| 多模态交互 | Step 3.7 Flash 拍照识别书架/区域 |
| 端到端闭环 | 前端上传/提问 → 编排调度 → 检索/规划/校验 → 回答，公网可访问 |

提交文档、部署说明、技术栈清单定稿——第七天，交卷。

省下的时间不会浪费：秋季开学前把测试数据切换为真实馆藏，让"吉小图"真正站上图书馆的服务台，才是这个项目真正的"决赛"。
