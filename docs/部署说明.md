# 部署说明文档

## 一、环境要求

### 硬件要求
- **DGX Spark 云节点**（推荐）：NVIDIA GPU，≥16GB 显存
- 或任意配备 NVIDIA GPU 的 Linux 服务器

### 软件要求
- Python 3.10+
- pip 或 conda
- Git

---

## 二、DGX Spark 云节点部署步骤

### Step 1: 激活实验云节点

1. 登录 DGX Spark 赛事平台，申请/激活实验云节点
2. 获取节点的 IP 地址和 SSH 访问凭证
3. SSH 连接到节点：
```bash
ssh username@<dgx-spark-ip>
```

### Step 2: 克隆项目

```bash
git clone <your-repo-url>
cd library-ai-guide
```

### Step 3: 安装依赖

```bash
# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装 Python 依赖
pip install -r requirements.txt

# 安装 NVIDIA 相关依赖（DGX Spark 上）
pip install tensorrt-llm  # 如平台已预装则跳过
```

### Step 4: 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，填入 Stepfun API Key 及其他配置
```

关键配置项：
- `STEPFUN_API_KEY`：阶跃星辰 API 密钥（必填）
- `STEPFUN_BASE_URL`：API 端点地址
- `NEMOTRON_MODEL_PATH`：本地 Nemotron 模型路径（可选，DGX Spark 上可能有预装路径）

### Step 5: 同步馆藏数据（可选但推荐）

```bash
# 从 MySQL 同步真实馆藏数据到知识库
python scripts/sync_mysql_to_kb.py
```

### Step 6: 构建 RAG 索引

```bash
python scripts/build_kb.py
```

### Step 7: 启动服务

```bash
# 开发模式（热重载）
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload

# 生产模式
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### Step 8: 验证部署

```bash
# 健康检查
curl http://localhost:8000/health

# 测试对话
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "图书馆开放时间是？"}'
```

## 三、本地开发环境部署

如果在本地 Windows/Mac 开发，步骤相同但注意：

1. **RAPIDS cuDF** 仅支持 Linux，本地开发时可跳过
2. **TensorRT** 仅支持 NVIDIA GPU，本地开发时可跳过
3. 系统会自动使用 CPU 模式运行 BM25 检索（KBM25Index 纯 Python 实现）
4. 本地运行时，知识库使用 JSON 文件中的模拟数据（已预置50条FAQ+25条规则+4层楼面数据）

## 四、前端访问

服务启动后：
- **本地访问**：`http://localhost:8000`
- **局域网访问**：`http://<服务器IP>:8000`
- **手机访问**：确保手机和服务器在同一网络，扫码或输入 URL 即可

前端支持移动端浏览器直接调用摄像头（需 HTTPS 或 localhost）。

## 五、常见问题

### Q: Step 3.7 Flash API 调用失败？
A: 检查 `.env` 中的 `STEPFUN_API_KEY` 是否正确。系统会自动降级到关键词匹配，不会完全不可用。

### Q: 知识库数据太旧？
A: 运行 `python scripts/sync_mysql_to_kb.py` 重新同步 MySQL 数据，然后运行 `python scripts/build_kb.py` 重建索引。

### Q: 相机权限被拒绝？
A: 浏览器需要 HTTPS 或 localhost 才能调用 `getUserMedia()`。如果是 HTTP 部署，点击"从相册选择"按钮使用备选上传方式。

### Q: DGX Spark 节点 GPU 不可用？
A: 系统会自动回退到 CPU 模式。所有核心功能（对话、RAG、导航）均可 CPU 运行，只是性能略有下降。

## 六、监控与日志

- 日志输出：控制台（彩色）+ `logs/` 目录（按日滚动）
- 健康检查：`GET /health` 返回服务状态、活跃会话数、知识库文档数
- 会话管理：`GET /session/{session_id}` 查看指定会话详情
