# 技术栈说明文档

## 概述

本项目严格遵循 DGX Spark Hackathon 平台适配性要求（评审权重15%），深度使用 NVIDIA SDK、NVIDIA 开源模型及 Stepfun 阶跃星辰模型。以下逐一说明每个技术组件的具体用途和使用方式。

---

## 一、Stepfun 阶跃星辰模型（硬性要求）

### Step 3.7 Flash — 多模态识别 + 轻量推理

| 使用场景 | 具体用途 | 技术参数 |
|----------|---------|---------|
| **多模态理解** | 用户上传照片 → Flash 原生视觉理解，识别书架标签、区域标识、楼层信息 | temperature=0.3 |
| **意图分类** | 7类用户意图精确分类（policy_query/location_where/navigate_to/book_lookup/opening_hours/facility_query/general_chat/image_query） | temperature=0.1 |
| **轻量回复生成** | 闲聊、问候等简单对话的快速响应 | temperature=0.8, max_tokens=200 |
| **事实校验** | Verify Agent 使用 Flash 对回复与知识库进行一致性比对 | temperature=0.1 |

**选择 Flash 版本的原因**：Flash 在保持多模态理解和低延迟的同时，成本更低。本项目将 Flash 专注于其核心优势——视觉理解和快速分类，而将重量级文本生成交给本地 GPU。

**深度使用体现**：
- 多模态 prompt 工程：为图书馆场景定制识别提示词（"观察书架标签/楼层标识/附近设施"）
- reasoning_effort 参数利用：low（闲聊/分类）→ medium（校验/查询重写）
- 混合路由架构：Flash 与本地模型协同工作，按任务类型智能分发

## 二、NVIDIA 平台

### DGX Spark GB10 — 统一内存架构

128GB 统一内存（CPU/GPU 共享），运行 Qwen3.6-35B-A3B MoE 4bit 量化模型。

| 指标 | 值 |
|------|:--:|
| GPU | NVIDIA GB10 |
| 统一内存 | 128GB |
| CUDA | 13.0 |
| 计算能力 | sm_120 (12.1) |
| 模型加载显存 | ~22GB |
| 推理速度 | ~43 tok/s |

### NVIDIA 开源模型 — Qwen3.6-35B-A3B MoE

| 使用场景 | 具体用途 |
|----------|---------|
| **本地文本生成** | DGX Spark 上通过 llama.cpp 进行 GPU 推理，处理图书馆咨询的回复生成 |
| **嵌入生成（计划）** | 用于 RAG 知识库的文本向量化 |

模型参数：
- 总参数：35B（MoE 稀疏激活）
- 激活参数：~3B/次推理
- 量化：MXFP4（4bit），GGUF 格式
- 上下文窗口：32K tokens

## 五、混合路由架构

```
用户请求
    │
    ▼
Orchestrator (意图分类 → Stepfun Flash)
    │
    ├── general_chat → Stepfun Flash (轻量快速, 保证身份)
    ├── 图片上传 → Stepfun Flash (多模态识别)
    ├── policy_query → BM25 RAG → Qwen3.6 MoE GPU (本地推理)
    ├── location_where → BM25 RAG → Qwen3.6 MoE GPU
    └── navigate_to → BM25 RAG → Planning Agent (A*) → Qwen3.6 MoE GPU
```

**路由规则**：
- 闲聊/问候 → Stepfun（保证"吉小图"身份一致性）
- 图片 → Stepfun 多模态（本地模型无视觉能力）
- 知识查询 → BM25 检索 → 本地 GPU 生成（展示 DGX Spark 算力）
- 意图分类 → Stepfun（快速，<1s）
- API 失败 → 自动降级 3 次重试 → 关键词搜索兜底
