## 五、往届获奖作品参考

### 作品1：高危化工巡检辅助Claw
- **团队**：中山大学软件工程学院 SYSU Gaming Team
- **场景**：高危化工巡检
- **技术方案**：
  - 基于DGX Spark构建化工场景知识库 + 混合RAG
  - 风险优先级规划 + 多模态数据融合
  - 结合OpenClaw与ROS实现巡检车交互控制
  - 异常触发告警 → 人工复核 → 生成报告
- **技术栈**：OpenClaw、ROS、多模态融合、RAG
- **借鉴点**：硬件+软件完整闭环，体现"知识库+RAG+多模态"的融合思路
- **项目链接**：https://github.com/xiaoZ857/NVIDIA_DGX_Spark_Hackathon

### 作品2：AI Agent 团队治理系统
- **团队**：Starfire-AgentTeam
- **场景**：多Agent统一管理调度
- **技术方案**：
  - 集成主流Agent框架
  - 解决多Agent协作混乱、权限不可控、记忆越界问题
  - 实现"有角色、有边界、有记忆"的组织化协作
- **借鉴点**：Agent架构设计深度，从"单兵作战"到"组织化协作"的核心理念
- **项目链接**：https://github.com/ZhanlinCui/Starfire-AgentTeam

### 共性规律总结

## 七、推荐方向（三个可选方案）

### 🔥 方向一：图书馆新生AI导览助手（⭐⭐⭐⭐⭐ 首选推荐）

**场景痛点**：每年新生入学，图书馆导览、借阅规则、座位预约等咨询量暴增，馆员重复回答效率低。

**核心思路**：
- 基于DGX Spark部署**Step 3.7 Flash**（原生多模态理解 + 高可靠工具调用）
- 构建图书馆知识库（馆藏分布、规章制度、FAQ）+ 混合RAG
- 多模态能力：拍照识别书架位置 → 返回路径规划；扫描图书封面 → 返回馆藏位置和借阅状态
- 多智能体协作：导览Agent（对话交互）+ 检索Agent（RAG查询）+ 规划Agent（路径生成）

**技术栈**：Step 3.7 Flash + NVIDIA NeMo + RAPIDS（加速检索）+ 轻量级前端

**亮点**：
- 解决真实痛点，有落地价值（评审25%）
- 体现Stepfun模型使用（评审硬性要求）
- 演示性强：手机拍照→识别→返回结果，观众一眼看懂
- 复杂度可控，适合10天完成

---

### 方向二：AI研学助手——长链路任务处理 + 报告生成（⭐⭐⭐⭐）

**场景痛点**：本科生写课程论文/文献综述时，需在多个数据库间跳转检索，流程繁琐、效率低。

**核心思路**：
- 主Agent负责意图理解 → 规划检索路径 → 调度子Agent执行 → 汇总生成报告
- Step 3.7 Flash擅长长程多轮工具调用（可稳定调用API/浏览器/终端）
- 接入图书馆馆藏系统API + 学术数据库（至少模拟接入一个）
- 关键节点人工确认（检索计划确认，避免方向跑偏）

**技术栈**：Step 3.7 Flash + MCP协议 + OpenClaw

**亮点**：
- 对标上一届"AI Agent团队治理系统"的架构深度
- 体现多智能体协同（评审25%）
- 长链路任务处理是生产级Agent的分水岭，易拉开差距

---

### 方向三：图书馆数字人荐书官——多模态 + 个性化推荐（⭐⭐⭐）

**场景痛点**：图书馆有大量借阅数据，但缺乏个性化推荐能力，用户找书难。

**核心思路**：
- Step 3.7 Flash原生多模态能力处理用户上传的图书封面/读书笔记截图
- 结合读者借阅历史（脱敏后）做个性化推荐
- 部署NVIDIA Nemotron做推理和推荐
- 集成轻量级数字人形象（开源3D avatar），增强演示效果

**技术栈**：Step 3.7 Flash + Nemotron + TensorRT（加速推理）

**亮点**：
- 视觉冲击力强，演示效果分高（10%）
- 体现NVIDIA开源模型使用（评审要求）
- 门槛相对较低，适合快速出demo
